# Построение безопасного пайплайна CI/CD для проекта DefectDojo

## 1. Введение
Настоящий документ описывает процесс построения безопасного пайплайна непрерывной интеграции и доставки (CI/CD) для open-source проекта DefectDojo в рамках выпускной квалификационной работы по профессии «Специалист по информационной безопасности» (трек DevSecOps).

Целью работы является внедрение практик безопасной разработки в виде статического и динамического анализа, анализа зависимостей, проверки конфигураций и секретов, а также создание шлюза безопасности для предотвращения развёртывания при обнаружении критических уязвимостей.

## 2. Архитектура пайплайна
Пайплайн реализован на базе облачного сервиса GitHub Actions и автоматически запускается при каждом коммите.

1. **Сборка (build):** установка зависимостей проекта и подготовка окружения.
2. **Статический анализ (SAST):** сканирование исходного кода на наличие уязвимостей с помощью Semgrep.
3. **Анализ зависимостей (SCA):** проверка сторонних библиотек и компонентов с помощью Trivy.
4. **Динамический анализ (DAST):** сканирование работающего веб-приложения с помощью OWASP ZAP.
5. **Проверки безопасности (Security Checks):** поиск секретов в репозитории с помощью GitLeaks и анализ конфигураций с помощью Trivy.
6. **Шлюз безопасности (Security Gateway):** анализ результатов проверок и остановка пайплайна при обнаружении критических уязвимостей.

## 3. Инструментарий и обоснование выбора
В ходе работы были выбраны и интегрированы следующие инструменты с открытым исходным кодом:

- **SAST (Semgrep):** инструмент статического анализа, выбранный за скорость работы и гибкость настройки правил.
- **SCA (Trivy):** инструмент для сканирования уязвимостей в зависимостях, поддерживающий экосистему Python и имеющий низкое потребление ресурсов.
- **DAST (OWASP ZAP):** инструмент динамического анализа, настроенный для сканирования приложения в тестовом окружении.
- **Security Checks (GitLeaks + Trivy):** GitLeaks используется для обнаружения секретов (паролей, токенов) в коде репозитория; Trivy в режиме `config` проверяет файлы конфигурации (например, Dockerfile) на наличие ошибок.
- **CI/CD (GitHub Actions):** облачный сервис, выбранный для автоматизации процессов благодаря простой интеграции с инструментами безопасности.

## 4. Реализация и результаты
Разработанный пайплайн позволяет на каждом этапе сборки автоматически выполнять проверки безопасности и предоставлять результаты в логах выполнения CI.

В результате проведённой настройки было обеспечено:
- Покрытие кода и работающего сервиса проверками на наличие распространённых уязвимостей.
- Автоматическая проверка зависимостей и конфигураций на каждом этапе разработки.
- Реализация механизма Security Gateway, который блокирует выполнение пайплайна в случае обнаружения критических ошибок на этапах SAST или SCA.

## 5. Заключение
В ходе выполнения дипломной работы был успешно разработан и внедрён безопасный пайплайн CI/CD для проекта DefectDojo. Система охватывает все ключевые этапы жизненного цикла разработки программного обеспечения, что позволяет на ранних стадиях выявлять и предотвращать потенциальные угрозы информационной безопасности.
